# 1. 프로젝트 개요

해당 프로젝트는 숭실대학교 학생혁신단 축제 부스 이벤트 웹페이지이다.
유형 테스트 웹페이지처럼 사용자에게 대략 7~8개 정도의 질문을 던진 뒤 사용자에게 맞는 유형을 반환해줘야한다.

# 2. 기능
### 2.1 기본적인 플로우. 질문을 던지고 이에 맞는 유형을 잘 던져줄 수 있어야한다.
### 2.2 질문 구성과 반환하는 유형은 html 혹은 js 파일과 분리되어 따로 하나의 문서로 관리할 수 있어야 한다. 즉 웹페이지에서 사용 시 문서에 있는 질문과 유형을 가져올 수 있어야 한다. 문서 유형은 txt 혹은 md 으로 한다.
### 2.3 인스타그램 스토리로 공유 기능이 있어야 한다.

# 3. 디자인
### 3.1 reference/main_reference.jpg 에서 디자인의 느낌을 차용한다. 즉 해당 이미지의 UI를 완벽히 따라가는 것이 아닌 폰트와 색상, border round의 정도 같은 느낌을 따라가야한다.
### 3.2 UI적인 구성과 User Flow는 타 유형 테스트 웹페이지와 비슷하게 한다.

# 6. 진행 현황

### 6.1 완료된 작업
- 개발 단계 기획 (Stage 1~7)
- `questions.md` 데이터 파일 형식 설계 및 임시 콘텐츠 작성 (유형 4개, 질문 8개)
- 전체 UI 구현 — 시작 화면 / 질문 화면 / 결과 화면
- 퀴즈 로직 구현 — questions.md 파싱, 점수 합산, 유형 반환
- 화면 전환 애니메이션 (슬라이드, 페이드)
- 파일 분리 리팩토링 (클린 아키텍처)

### 6.2 파일 구조
```
├── index.html        — HTML 구조
├── questions.md      — 질문·유형 데이터 (콘텐츠 확정 후 교체)
├── css/
│   └── style.css     — 전체 스타일
└── js/
    ├── parser.js     — questions.md 파싱 (순수 함수)
    ├── store.js      — 퀴즈 상태 관리
    ├── router.js     — 화면 전환
    ├── renderer.js   — DOM 렌더링
    └── main.js       — 진입점, 이벤트 바인딩
```

### 6.3 미완료 작업
- Stage 1 : 실제 질문·유형 콘텐츠 확정 후 questions.md 교체
- Stage 6 : 인스타그램 스토리 공유 기능 구현
- Stage 7 : 결과 화면 캐릭터 이미지 교체, 호스팅 배포
